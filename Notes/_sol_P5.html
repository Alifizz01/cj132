P5-1. Compute a section's bus V / I / P. Apply the §5.2 rule at each level — series sums volts and keeps current common; parallel sums amps and keeps voltage common.
(a) One string (12 cells in SERIES):  voltages add ⇒ V = 12 × 2.30 V = 27.6 V; subtract the one series blocking diode (≈0.7 V comes off the voltage) ⇒ 26.9 V. Current is common ⇒ I = one cell's 0.50 A = 0.50 A (unchanged by the series diode).
(b) Section voltage (16 strings in PARALLEL):  voltage is common across parallel branches ⇒ Vsection ≈ 26.9 V (it does not multiply by 16).
(c) Section current:  currents add ⇒ Isection = 16 × 0.50 A = 8.0 A.
(d) Section power:  P = V·I = 26.9 V × 8.0 A = 215.2 W ≈ 215 W.
Summary of which operation applied: step (a) voltage = added (series), step (a) current = common (series); step (b) voltage = common (parallel); step (c) current = added (parallel). The single most common slip is multiplying the voltage by 16 in (b) — that would break "voltage is common in parallel."
P5-2. Which diode protects, and why. Tell them apart by placement: blocking = in series with a string, stops external back-feed; bypass = across a cell, stops internal reverse-bias/hotspot.
(a) Battery pushing current back into the dark array at night → BLOCKING diode. It is in series at each string's output and lets current out but blocks current trying to flow back into the array, so the battery cannot dump current into the cells in eclipse. (External back-feed.)
(b) One shaded cell partway up a string overheating → BYPASS (shunt) diode. The common string current forces the shaded cell into reverse bias; the bypass diode across it turns on at ≈0.7 V, carries the string current around the bad cell and clamps it to ≈−0.7 V, capping the hotspot. (Internal reverse bias.)
(c) Brighter parallel neighbours driving current down into a cold mismatched string → BLOCKING diode. Because voltage is common on the shared rails, the bright strings can back-feed the weak one; the blocking diode at the weak string's output refuses that reverse current. (External back-feed, between strings.)
Pattern: (a) and (c) are both external current trying to enter a string → blocking; (b) is an internal cell being reverse-biased → bypass.
P5-3. Sizing, plus a reference-break trap.
(a) Series ⇒ volts add, so n ≈ target ÷ cell voltage = 50 V ÷ 2.30 V ≈ 21.7. Round up to 22 cells (you cannot fit a fractional cell, and rounding down to 21 gives 21 × 2.30 = 48.3 V, which falls short of 50 V). Check: 22 × 2.30 V = 50.6 V, just clearing 50 V before the blocking-diode drop and any margin for the hot/end-of-life worst case.
(b) What went wrong: string.fromSingleCell(...) takes a deep copy of the prototype cell at the moment it is called (it clones the cell 12 times). The later cell.setDose(20,20) modifies only the original prototype, which is now independent of the 12 copies inside the string — so the string is still made of zero-dose ("brand-new") cells. This is the reference break (§5.5).
Fix: either set the dose on the prototype before building (cell.setDose(20,20) first, then string.fromSingleCell(...)), or set it on the string so the cascade reaches every copy (s.setDose(20,20) — a forwarding command, §5.4). Both give all 12 internal cells the intended dose.
